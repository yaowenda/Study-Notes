## 应急响应整体思路

### 1、发现异常

挖矿病毒常表现为：

- CPU、内存资源长期高占用；
- 网络带宽异常增大；
- 任务管理器中出现不明进程；
- 系统运行卡顿、风扇高转速；
- 安全日志中存在大量登录尝试或异常账号行为。

可以通过任务管理器定位异常网络的位置，先找到其位置而不是先关闭进程，好留存病毒后面分析

### 2、终止恶意进程与隔离系统

- 在任务管理器中结束可疑的挖矿进程；
- 禁止病毒文件所在目录的执行权限；
- 暂时断开网络，以防止远控或继续传播；
- 若为服务器环境，应立即从负载中摘除受感染主机。

### 3、病毒样本分析（研究阶段）

逆向或代码审计，监控病毒运行时行为（动态分析）

判断是否包含：

- 自启动机制；
- 持久化后门；
- 数据窃取模块；
- 挖矿程序（如xmrig、nicehash）

### 4、判断病毒入侵途径

病毒（或后门）之所以能进入系统，通常有以下几种原因：

| 入侵途径           | 说明                                             |
| ------------------ | ------------------------------------------------ |
| **账号凭据泄露**   | 弱口令、暴力破解、钓鱼、社工等方式被攻击者获取。 |
| **系统漏洞利用**   | RCE（远程代码执行）、内核漏洞、权限提升漏洞等。  |
| **第三方应用漏洞** | 未更新的CMS、OA、数据库管理工具等。              |
| **网站/服务漏洞**  | SQL注入、文件上传漏洞、任意文件写入等。          |

### 5、检查可疑账号与后门

攻击者往往会通过创建隐藏账户保持控制：

- 在「此电脑 → 管理 → 本地用户和组 → 用户」中查看；

- 以 `$` 结尾的用户名通常为隐藏账号；

- 或者使用`windowslog.exe`工具查看，更方便，在`D:\蓝队应急响应工具箱v2025.3\Tools\BT-Log\windows1check`

  

  **若攻击者是通过漏洞进入系统，则常会新建账户，为什么要创建账号？后续可以通过远程桌面访问；
   若是暴力破解管理员账号进入，则无需新建账户，没必要了。**

**处置措施：**

- 修改管理员账号密码；
- 删除隐藏账户；
- 检查远程桌面登录策略；
- 禁止3389端口对外暴露。

### 6. 日志分析与入侵验证

重点分析以下内容：

- **安全日志（Event Viewer → Windows Logs → Security）**：暴力破解尝试、登录失败次数；
- **系统日志**：异常服务启动、账户创建；
- **IIS/Apache/Nginx日志**：同一 IP 对多个目录访问、404/302 比例高，可能为扫描行为；
- **文件时间戳**：可疑文件创建时间是否集中且接近事件发生时间。

### 7. 系统恢复与加固

**恢复阶段：**

- 清除病毒残留文件；
- 修复系统关键服务；
- 更新补丁、升级软件；
- 重启并监控CPU、网络等资源恢复情况。

**加固阶段：**

- 关闭不必要的端口和服务；
- 启用防火墙与安全审计；
- 部署主机入侵检测（HIDS）或EDR系统；
- 设置复杂密码策略与多因素认证；
- 定期备份重要数据。
